# LogStream - Distributed Log Aggregation and Monitoring System

## Project Overview

LogStream is a distributed log aggregation and monitoring system designed to collect, process, store, and visualize logs generated by multiple applications and services.

The system centralizes logs from different sources, enriches them with metadata, stores them in MongoDB, and provides a dashboard for monitoring, filtering, searching, and analyzing logs.

---

## Problem Statement

Modern distributed applications generate large volumes of logs across multiple services and servers. Managing these logs separately makes debugging, monitoring, and troubleshooting difficult.

The objective of LogStream is to provide a centralized platform for:

* Log Collection
* Log Processing
* Log Storage
* Log Search and Filtering
* Log Monitoring and Analytics

---

## Proposed Solution

The proposed solution aggregates logs from multiple sources into a centralized system.

The system performs the following operations:

1. Receive logs through a Flask API.
2. Process and enrich logs with metadata.
3. Store logs in MongoDB.
4. Provide real-time visualization using Streamlit.
5. Support filtering, searching, and analytics.

## CAP Theorem

LogStream follows the AP (Availability + Partition Tolerance) model.

Reasons:

* Continuous log collection is critical.
* System should remain available during network failures.
* Minor delays in log synchronization are acceptable.
